# Implementation Plans

Use this folder for PRDs, technical designs, and implementation plans before meaningful code changes.

## Recommended flow

1. `YYYY-MM-DD-feature-brief.md` — product brief / PRD.
2. `YYYY-MM-DD-feature-design.md` — technical design.
3. `YYYY-MM-DD-feature-plan.md` — bite-sized implementation plan.

## Plan quality bar

A useful plan includes:

- goal and non-goals,
- exact files to inspect or change,
- acceptance criteria,
- test strategy,
- rollout/rollback notes if production-facing,
- small tasks that can be implemented and reviewed independently.
