## Test strategy

Use this hierarchy as the project matures:

1. **Unit tests:** pure functions, parsers, reducers, validation logic.
2. **Integration tests:** API routes, persistence, background jobs, external-service adapters with fakes.
3. **End-to-end tests:** core user journeys only, because they are slower and more brittle.
4. **Regression tests:** every production bug fix should add a test that would have caught the bug.

## Reporting standard

Every PR should include:

- summary of user-visible change,
- test commands run,
- screenshots/log excerpts when useful,
- known risks or skipped checks.
